### CRITIQUE

Based on the experiments and the original prompt template, here are specific areas for improvement:

- **Add explicit stage detection logic** - The prompt relies on implicit stage transitions. Add clear conditional checks like: "If data exists but no analysis exists → proceed to analysis stage"

- **Strengthen tool restriction enforcement** - Current tool prohibitions are descriptive but lack explicit blocking mechanisms. Add validation checks before tool execution.

- **Define clearer completion criteria for each stage** - The prompt doesn't specify what constitutes "finished" for data collection or analysis. Add measurable completion standards.

- **Improve state tracking visibility** - Add explicit status reporting at each transition: "Stage: Data Collection → Complete" to prevent redundant operations.

- **Standardize output formats per stage** - Beyond the examples, enforce strict templates: `[DATA_COLLECTION] {content}` and `[ANALYSIS] {content}`

- **Add progress validation checks** - Implement explicit verification like: "Before analysis, verify data exists; before termination, verify analysis exists"

- **Clarify error recovery procedures** - Specify exact remediation steps for violations rather than generic "re-generate" instructions.
