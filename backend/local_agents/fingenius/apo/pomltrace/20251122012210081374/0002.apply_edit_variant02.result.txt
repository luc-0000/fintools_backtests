### CRITIQUE

**Critique:**

*   **Missing Explicit State Tracking:** The prompt relies on the model to infer the "current stage" from the context, but it does not provide a clear, structured mechanism for the model to identify or confirm which stage it is in. This leads to ambiguity and stage-transition errors.
*   **Vague "Current State" Reference:** The instruction "根据当前状态" ("based on the current state") is too ambiguous. The prompt fails to specify what constitutes the "state" (e.g., the presence of specific data keys in the context, a previous analysis output, or a user message), leaving the model to guess.
*   **Poorly Defined Stage-Transition Logic:** The rule "如果状态中已经含有收集的数据，不要重复收集，进入下一个流程!" is a good idea but is implemented as a footnote. The core logic for progressing through the workflow is not the primary directive, leading the model to fixate on the per-stage rules without understanding the overarching goal of progression.
*   **Ineffective Conditional Feedback:** The `{{ if }}` block is a post-hoc check for a specific violation. It does not proactively guide the model on *how* to correctly transition between stages, only that it has done so incorrectly after the fact. This is corrective rather than preventative.
