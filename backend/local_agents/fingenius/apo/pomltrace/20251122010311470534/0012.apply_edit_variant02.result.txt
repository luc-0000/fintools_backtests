### CRITIQUE

**Critique:**

*   **Missing Explicit State Transition Logic:** The prompt tells the agent to "judge which processes have been completed" and "directly execute the next process," but it provides no concrete rules or examples for *how* to make that judgment. The agent is left to guess when to move between stages.
*   **Ambiguous "State" Definition:** The prompt relies on a `{当前阶段}` variable, but the core instruction ("if the state already contains collected data, do not collect again") implies the agent must also inspect the *content* of the state. This dual source of truth (variable vs. content) is confusing.
*   **Poorly Integrated Conditional Logic:** The `{{ if 输出包含 "terminate" ... }}` check is a reactive penalty, not a proactive guide. A well-designed prompt should prevent the error from occurring in the first place by making the rules unmistakable.
*   **Vague "Important Prompt":** The final instruction ("if state already has data, don't collect") is a critical rule buried at the end. It should be a primary, reinforced constraint within each stage's definition, especially in the Data Collection phase.
