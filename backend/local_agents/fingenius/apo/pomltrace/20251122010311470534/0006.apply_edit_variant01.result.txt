# Prompt Template

根据当前状态，判断哪些流程已经执行完毕，直接执行下一个流程。如果流程前面已经执行过，不要重复执行。

**工作流程规则**：
1. **数据收集阶段**：  
   - 允许使用 `数据收集工具`，**禁用其他所有工具**（包括 `terminate`）。  
   - 指令示例：  
     > "当前仅允许使用数据收集工具，禁止调用任何其他工具。输出格式：`[数据收集结果]`"

2. **深度分析阶段**：  
   - **禁用所有工具**（包括 `terminate`），仅允许生成分析文本。  
   - 指令示例：  
     > "当前处于深度分析阶段，禁止调用任何工具。请基于数据生成专业分析报告，不要解释思考过程。"

3. **综合结论阶段**：  
   - 允许调用 `terminate` 结束任务。  
   - 指令示例：  
     > "分析完成后，调用 `terminate` 工具提交最终结论。"

# Critique

Based on the experiments, here are specific issues and testable improvements:

- **Clarify stage transition logic** - Add explicit conditions for moving between stages (e.g., "If data exists → skip to analysis; if analysis complete → skip to conclusion")
- **Strengthen tool restriction enforcement** - Make prohibitions more absolute: "Under NO circumstances may any tool except data collection be used in stage 1"
- **Improve error handling specificity** - Replace generic违规操作 with stage-specific instructions: "Data collection stage violation: Remove terminate call and proceed with analysis using collected data"
- **Add completion criteria** - Define what constitutes "finished" for each stage (e.g., "Data collection complete when [specific data types] obtained")
- **Standardize output formats** - Require explicit stage markers in responses: "[STAGE: DATA_COLLECTION] [DATA: ...]" to prevent tool misuse
- **Handle partial progress** - Add logic for when some but not all data exists: "If partial data exists, proceed to analysis noting gaps"
- **Prevent premature termination** - Add checkpoint: "Before calling terminate, verify analysis section meets minimum length/coverage requirements"
