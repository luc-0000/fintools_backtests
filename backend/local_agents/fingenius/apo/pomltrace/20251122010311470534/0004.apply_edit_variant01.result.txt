# Critique

Based on the provided experiments, here are specific critiques and testable improvements for the prompt template:

- **Add explicit stage detection logic**: The template lacks clear rules for determining current stage. Add conditional checks like: `{{ if 状态包含数据收集结果 }}进入深度分析阶段{{ else if 状态包含分析报告 }}进入综合结论阶段{{ else }}数据收集阶段{{ end }}`

- **Enforce strict output format per stage**: Current examples are vague. Specify exact formats: Data collection must output `[DATA_COLLECTION_RESULT: {data}]`, analysis must output `[ANALYSIS_REPORT: {text}]`, conclusion must call `terminate` with `[FINAL_CONCLUSION: {summary}]`

- **Prevent stage regression**: Add constraint `{{ if 前一阶段已完成 }}禁止返回前一阶段操作{{ end }}` to ensure linear progression through stages

- **Clarify data reuse rules**: Replace vague "不要重复收集" with explicit check: `{{ if 状态.数据收集结果存在 }}跳过数据收集，直接分析{{ end }}`

- **Add completion criteria per stage**: Define clear completion signals: data collection ends when `[DATA_COLLECTION_RESULT]` appears, analysis ends when `[ANALYSIS_REPORT]` appears

- **Improve error messaging**: Make violations more specific: `❗ 数据收集阶段违规：检测到非数据收集工具调用。允许工具：数据收集工具。检测到工具：{违规工具名}`

- **Standardize stage identifiers**: Use consistent stage labels (`"数据收集"`, `"深度分析"`, `"综合结论"`) throughout template instead of mixing terms
