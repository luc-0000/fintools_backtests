### CRITIQUE

**Critique of the Prompt**

**1. Structural Issues**  
- **Missing goal**: The prompt lacks a clear, one-line success definition (e.g., “Produce a final conclusion via `terminate` after data collection and analysis”).  
- **Circular dependencies**: The “judge which processes are complete” step depends on undefined “current state,” creating ambiguity in progression logic.  
- **No stop condition**: The stop condition is only implied (calling `terminate` in the final stage), but no explicit completion check exists for earlier stages.

**2. Instruction Quality**  
- **Vague verbs**: “Judge which processes have been executed” is ambiguous—no criteria or state variables are defined.  
- **Mixed abstraction**: High-level workflow rules are mixed with implementation examples, causing potential confusion.  
- **Overlapping scope**: The “don’t repeat processes” rule appears in both the introduction and “Important Tip,” creating redundancy.

**3. Control and Behavior**  
- **Unclear uncertainty handling**: No guidance on how to proceed if the “current state” is missing or unclear.  
- **Verbosity confusion**: Some stages demand pure text output, but the prompt includes example outputs that may encourage unnecessary formatting.  
- **Feedback omission**: No requirement to report stage transitions or confirm state before proceeding.

**4. Input and Output Specification**  
- **No input defaults**: “Current state” is undefined—what if no state is provided?  
- **Output schema missing**: Only example formats are given; no strict schema is enforced for stages.  
- **Format inconsistency**: Output format changes per stage (e.g., `[数据收集结果]` vs. free text), but rules for switching are not explicit.

**5. Scope and Safety**  
- **No error handling**: No instructions for tool failures or invalid state transitions.  
- **User authority ambiguity**: The model must infer state without user confirmation, risking incorrect stage jumps.

**6. Efficiency and Maintainability**  
- **Redundancy**: “Don’t repeat processes” is stated multiple times.  
- **Hidden assumptions**: Assumes “current state” will clearly indicate completed stages.

**Summary**: The prompt defines a three-stage workflow but lacks clarity on state management, progression logic, and error handling. It needs explicit state variables, conflict resolution rules, and consistent output schemas to ensure reliable execution.
