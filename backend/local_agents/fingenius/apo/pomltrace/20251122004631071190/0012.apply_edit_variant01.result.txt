# Critique

**Critique of the Prompt**

**1. Structural Issues**  
- **Missing goal**: The prompt lacks a clear, one-line objective. It describes stages but not the overall task success condition.  
- **No stop condition**: The only stop is via `terminate`, but the trigger for moving to the final stage is ambiguous—no explicit completion criteria for earlier stages.  
- **Circular dependency risk**: The model must infer which stage is current based on state, but state transitions are not clearly defined.

**2. Instruction Quality**  
- **Vague verbs**: “判断哪些流程已经执行完毕” (judge which processes are complete) is subjective—no measurable criteria.  
- **Mixed abstraction**: High-level rules (“不要重复执行”) are mixed with tool-specific constraints, creating confusion.  
- **Overlapping scope**: The “重要提示” repeats earlier instructions about not repeating data collection.

**3. Control and Behavior**  
- **Unclear uncertainty handling**: No guidance if the current stage is ambiguous—should the model ask or assume?  
- **Verbosity confusion**: Some stages demand pure text; others require specific output formats, but transitions are not managed.  
- **Feedback omission**: No requirement for progress reporting between stages.

**4. Input and Output Specification**  
- **No input defaults**: What if “当前状态” is missing or incomplete?  
- **Format inconsistency**: Output format changes per stage (e.g., `[数据收集结果]` vs. free text), but the model must track this without clear state signals.  
- **No validation**: No step to verify stage completion before proceeding.

**5. Scope and Safety**  
- **User authority ambiguity**: The model autonomously decides stage transitions, risking premature termination or repetition.  
- **No error handling**: If a tool fails or data is invalid, recovery steps are missing.

**6. Efficiency and Maintainability**  
- **Redundancy**: Multiple reminders not to repeat actions.  
- **Hidden assumptions**: Assumes the model can accurately track “current stage” without explicit state input.

**Summary**: The prompt defines stages but fails to provide clear state management, completion criteria, or error handling. It risks inconsistent behavior due to vague transitions and overlapping instructions.
