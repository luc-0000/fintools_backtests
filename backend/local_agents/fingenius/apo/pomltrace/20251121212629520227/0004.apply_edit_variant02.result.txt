### CRITIQUE

**Critique of Prompt**

**1. Structural Issues:**  
- **Missing goal:** The prompt lacks a clear, one-line success definition (e.g., "Produce a final conclusion using collected data").  
- **No stop condition:** The only stop is via `terminate`, but conditions for reaching the final stage are vague.  

**2. Instruction Quality:**  
- **Vague verbs:** "判断哪些流程已经执行完毕" (judge which processes are complete) is ambiguous—no criteria provided.  
- **Mixed abstraction:** High-level workflow rules are mixed with implementation examples, creating potential confusion.  

**3. Control and Behavior:**  
- **Unclear uncertainty handling:** No guidance on how to proceed if stage transitions are ambiguous or data is incomplete.  
- **Verbosity confusion:** Some stages demand concise outputs; others require detailed reports without clear differentiation.  

**4. Input and Output Specification:**  
- **No input defaults:** How to handle missing state information (e.g., undefined "current stage") is unspecified.  
- **Format inconsistency:** Output formats differ per stage (`[数据收集结果]` vs. free-text reports), but validation is only applied conditionally.  

**5. Scope and Safety:**  
- **User authority ambiguity:** The model must infer stage transitions autonomously, risking incorrect tool usage (e.g., calling `terminate` prematurely).  
- **No error handling:** No instructions for retries or handling invalid tool calls beyond a conditional warning.  

**6. Efficiency and Maintainability:**  
- **Redundancy:** Rules and examples are repeated across stages.  
- **Hidden assumptions:** Assumes the model can accurately track state and stage progress without explicit state parameters.  

**Summary:** The prompt defines a multi-stage workflow but suffers from ambiguous transitions, insufficient state management, and unclear completion criteria. It needs explicit stage-transition logic, error handling, and a unified output schema.
