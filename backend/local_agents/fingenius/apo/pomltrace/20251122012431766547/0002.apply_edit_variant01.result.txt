# Prompt Template

根据当前状态，判断哪些流程已经执行完毕，直接执行下一个流程。如果流程前面已经执行过，不要重复执行。

**工作流程规则**：
1. **数据收集阶段**：

# Critique

**Critique:**

*   **Missing Explicit State Tracking:** The prompt relies on the LLM to infer the "current stage" from context, but it does not provide a clear, machine-readable signal (like a state variable in the system prompt) to indicate which phase is active. This leads to ambiguity.
*   **Ambiguous Phase Transition Logic:** The rule "if state already has collected data, proceed to the next stage" is vague. It doesn't specify *how* the agent should check for this data or what constitutes "enough" data to trigger the transition, leaving the transition logic open to flawed interpretation.
*   **Conflicting "Do Not Repeat" Instructions:** The core instruction "do not repeat executed processes" is given high priority but conflicts with the phase-specific rules. An agent might incorrectly prioritize avoiding repetition over following the tool-use restrictions of a given phase.
*   **Insufficient Guardrails for Tool Use:** The conditional check `{{ if 输出包含 "terminate" ... }}` is a good start but is reactive rather than preventative. The prompt lacks proactive, phase-specific instructions that are enforced *before* the LLM generates a response containing a forbidden tool call.
