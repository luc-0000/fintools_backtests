# Critique

Based on the experiments showing successful outcomes but varying reward scores (0.25 to 0.77), here are specific, testable improvements to optimize the prompt template:

- **Add explicit stage detection logic** - Include clear conditional statements that check for existing data in the state and explicitly transition between stages rather than relying on implicit understanding

- **Implement stricter output validation** - Add regex patterns or keyword checks to validate that each stage's output format is strictly followed (e.g., `^\[数据收集结果\]` for data collection)

- **Enforce stage progression constraints** - Add explicit guards that prevent reverting to previous stages once data is collected or analysis is generated

- **Standardize reward signal interpretation** - Since rewards vary significantly (0.10 to 0.77), add specific criteria for what constitutes optimal performance at each stage to maximize scoring consistency

- **Clarify tool usage boundaries** - Make the tool prohibition rules more explicit with negative examples showing exactly what constitutes a violation in each phase

- **Add state tracking variables** - Include explicit stage markers in the state that must be updated and checked to prevent stage repetition or skipping

- **Improve error recovery guidance** - Add specific remediation steps for when violations occur, rather than generic warnings about rule breaches
