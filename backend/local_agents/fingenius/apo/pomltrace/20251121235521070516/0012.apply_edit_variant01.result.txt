# Critique

**Critique of the Prompt**

**1. Structural Issues**  
- **Missing goal**: The prompt lacks a clear, one-line objective. It describes stages but not the overall purpose (e.g., "Analyze data to produce a final conclusion").  
- **No stop condition**: The only explicit stop is via `terminate`, but it’s unclear when to trigger it beyond "analysis completed."  
- **Circular dependency risk**: The rule "if data exists, skip collection" depends on undefined "current state," creating ambiguity.

**2. Instruction Quality**  
- **Vague verbs**: "Generate professional analysis report" lacks specificity—what qualifies as "professional"?  
- **Mixed abstraction**: High-level rules (e.g., "judge completed processes") mix with implementation details (e.g., tool restrictions).  
- **Overlapping scope**: The "skip if already done" rule appears in both the preamble and stage 1, causing redundancy.

**3. Control and Behavior**  
- **Unclear uncertainty handling**: No guidance on how to proceed if data collection fails or if the state is ambiguous.  
- **Verbosity confusion**: Stage 2 demands "no thinking process," but the prompt itself includes explanatory examples.  
- **No feedback mechanism**: Missing progress reporting between stages (e.g., confirming stage transitions).

**4. Input and Output Specification**  
- **No input defaults**: The "current state" is undefined—what if no data exists? Assumptions are implicit.  
- **Output schema inconsistency**: Stage 1 requires `[数据收集结果]`, stage 2 expects free-text analysis, and stage 3 uses `terminate`. Format shifts disrupt flow.  
- **No validation**: No step to verify analysis quality before calling `terminate`.

**5. Scope and Safety**  
- **Scope creep**: "Generate professional analysis" is open-ended—could lead to irrelevant content.  
- **User authority ambiguity**: The model autonomously transitions stages without user confirmation.  
- **No error handling**: Tool failures or invalid data aren’t addressed.

**6. Efficiency and Maintainability**  
- **Redundancy**: The "don’t repeat" rule is stated multiple times.  
- **Hidden assumptions**: Assumes "current state" tracks stage and data implicitly.  
- **Poor auditability**: Lack of section labels (e.g., `#policy`, `#procedure`) makes the prompt harder to debug.

**Summary**: The prompt defines a staged workflow but suffers from ambiguous state management, undefined goals, and inconsistent output formats. Clarify the end goal, define state explicitly, and standardize transitions and error handling.
