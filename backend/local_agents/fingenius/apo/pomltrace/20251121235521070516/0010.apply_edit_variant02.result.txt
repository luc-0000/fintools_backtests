### CRITIQUE

Based on the experimental results, here are specific critiques and testable improvements for the prompt template:

- **Add explicit stage transition triggers**: The template lacks clear conditions for moving between stages. Add explicit triggers like "When data collection returns [数据收集结果], immediately transition to deep analysis phase" to prevent stage confusion.

- **Define clearer data collection completion criteria**: Specify what constitutes valid "[数据收集结果]" format and content requirements to prevent premature or delayed stage transitions.

- **Strengthen tool restriction enforcement**: Add explicit rejection phrases for incorrect tool usage, such as "Tool not allowed in current phase. Current phase only permits: [allowed tools]" when violations occur.

- **Improve state tracking visibility**: Include explicit phase markers in the output format (e.g., "[Phase: Data Collection] [数据收集结果]") to make stage transitions more observable and verifiable.

- **Add error recovery guidance**: Include specific instructions for handling violations, such as "If tool call is rejected, regenerate response using only permitted methods for current phase" to improve robustness.

- **Clarify terminate usage conditions**: Explicitly state that terminate is ONLY permitted after analysis text has been generated, not just when reaching the conclusion phase.

- **Standardize output format per phase**: Define required output formats for each phase (e.g., "Analysis Report: [content]" for deep analysis) to reduce ambiguity in phase identification.
