### CRITIQUE

**Critique:**

*   **Missing Explicit State Definition:** The prompt does not define a clear, machine-readable format for the "current state." The agent must infer the stage from the conversation history, leading to inconsistent stage transitions.
*   **Ambiguous Stage Transition Logic:** The rule "如果状态中已包含有效收集数据，跳过此阶段" is too vague. The agent is left to interpret what constitutes "有效收集数据," causing it to potentially skip the data collection stage prematurely or get stuck.
*   **Poorly Defined Output Validation:** The conditional check `{{ if 输出包含 "terminate" ... }}` is a reactive, post-hoc measure. The prompt lacks proactive, positive instructions that explicitly forbid tool use in the analysis stage *before* the agent attempts it.
*   **Conflicting Instructions on Tool Use:** The instruction "直接执行下一个流程" (directly execute the next step) in the opening sentence, combined with the lack of a clear state, may encourage the agent to use available tools (like `terminate`) to "execute" the final step, even when it's prohibited.
