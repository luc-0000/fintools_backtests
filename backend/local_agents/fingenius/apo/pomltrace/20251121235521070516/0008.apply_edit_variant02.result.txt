### CRITIQUE

**Critique of the Prompt**

**Structural Issues:**  
- **Missing goal:** The prompt lacks a clear, one-line success definition (e.g., “Produce a final conclusion via terminate after valid data collection and analysis”).  
- **No stop condition:** Completion is only implied by reaching the “comprehensive conclusion” stage and calling `terminate`.  
- **Circular dependency risk:** The rule “If data is already collected, skip to next stage” could conflict with stage-specific tool restrictions if state-tracking is ambiguous.

**Instruction Quality:**  
- **Vague verbs:** “Judge which processes are completed” is subjective—no criteria are given for determining completion.  
- **Mixed abstraction:** High-level workflow rules are mixed with example instructions, creating redundancy.  
- **Overlapping scope:** Multiple sections reiterate tool restrictions; these could be consolidated into a single tool policy table.

**Control and Behavior:**  
- **Unclear uncertainty handling:** No guidance on what to do if the current stage is unknown or if data appears incomplete.  
- **Verbosity confusion:** Example outputs specify brevity, but the prompt doesn’t enforce this uniformly.  
- **No feedback mechanism:** The model isn’t instructed to announce stage transitions or validate inputs.

**Input and Output Specification:**  
- **No input defaults:** If “current state” is undefined, behavior is unspecified.  
- **Format inconsistency:** Example outputs use different formats (`[数据收集结果]` vs. free text); the final output format for `terminate` is undefined.

**Scope and Safety:**  
- **No error handling:** Tool failures or invalid data aren’t addressed.  
- **User authority ambiguity:** The model autonomously decides stage transitions without user confirmation.

**Efficiency and Maintainability:**  
- **Redundancy:** Tool restrictions are repeated in each stage; a single tool policy section would be clearer.  
- **Hidden assumptions:** Assumes the model can accurately track “current stage” and “already executed” processes without explicit state variables.

**Overall:** The prompt defines a staged workflow but lacks clarity in state management, completion criteria, and error handling, risking inconsistent execution.
