### CRITIQUE

**Critique of the Prompt**

**Structural Issues:**  
- **Missing goal:** The prompt lacks a clear, one-line success definition (e.g., “Produce a final analysis conclusion from data”).  
- **Contradictions:** The “terminate” tool is disabled in early stages but required for final submission—this is clear, but the conditional check (`{{ if ... }}`) is ambiguous in execution context.  
- **No stop condition:** The task ends only via `terminate` in the final stage, but no fallback is defined if `terminate` fails.

**Instruction Quality:**  
- **Vague verbs:** “判断哪些流程已经执行完毕” (“judge which processes are complete”) is subjective without explicit completion markers.  
- **Mixed abstraction:** High-level rules (e.g., “avoid repeating completed processes”) mix with low-level tool restrictions, creating potential conflicts.  
- **Overlapping scope:** State validation is mentioned in both “状态处理规则” and the final “重要提示,” causing redundancy.

**Control and Behavior:**  
- **Unclear uncertainty handling:** No guidance if state is ambiguous—defaults to data collection, but what if analysis data exists but is incomplete?  
- **Verbosity confusion:** Analysis stage demands “不少于300字” (≥300 words), but other outputs are terse—style isn’t consistently defined.  
- **Feedback omission:** No progress preamble is required before multi-step execution, risking user confusion.

**Input and Output Specification:**  
- **No input defaults:** Unclear what “数据不完整” (“incomplete data”) means—no validation criteria are provided.  
- **Format inconsistency:** Output formats shift between `[数据收集结果]`, pure text, and tool calls.  
- **No validation:** Missing a “verify before submit” step for the final conclusion.

**Scope and Safety:**  
- **No error handling:** If a tool fails, the prompt says “output error and stay in current stage,” but recovery steps are undefined.  
- **User authority ambiguity:** The model autonomously judges completed stages without user confirmation for irreversible actions.

**Efficiency and Maintainability:**  
- **Redundancy:** Multiple reminders to check state and avoid repetition.  
- **Hidden assumptions:** Assumes the model can accurately track “current stage” across turns with no memory mechanism described.

**Summary:** The prompt enforces a staged workflow but suffers from ambiguous state transitions, undefined error handling, and inconsistent output formatting. Clarify completion criteria, state validation rules, and error recovery to improve reliability.
