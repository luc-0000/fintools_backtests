# Critique

Based on the provided experiments, here are specific critiques and testable improvements for the prompt template:

- **Add explicit state tracking variables** - Include placeholders like `{{current_stage}}` that must be updated after each phase completion to prevent stage confusion
- **Define clearer phase transition triggers** - Specify exact conditions for moving between stages (e.g., "when data collection output exceeds 200 characters, proceed to analysis")
- **Strengthen tool restriction enforcement** - Add negative examples showing rejected tool usage with explicit error messages for each prohibited scenario
- **Include data validation checks** - Add template logic like `{{ if collected_data and length(collected_data) < minimum_requirement }}` to prevent premature stage advancement
- **Standardize output format per phase** - Require specific markers (e.g., `[DATA_COLLECTION_END]`) to explicitly signal phase completion rather than implicit content detection
- **Add progress confirmation steps** - Insert verification prompts like "Confirm all required data points collected? [YES/NO]" before allowing phase transitions
- **Clarify terminate conditions** - Specify exact content requirements before terminate is permitted (e.g., "must contain 3 key findings and 1 recommendation")
- **Improve error recovery guidance** - Add fallback instructions for when stages are incorrectly skipped, with explicit rollback procedures
