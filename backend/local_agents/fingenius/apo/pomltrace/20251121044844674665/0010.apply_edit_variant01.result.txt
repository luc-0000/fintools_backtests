# Critique

**Critique of the Prompt**

**Structural Issues:**  
- **Missing goal:** The prompt lacks a clear, one-line success definition (e.g., "Produce a final conclusion after data collection and analysis").  
- **Contradictions:** The rule "If data is already collected, skip to analysis" conflicts with the rigid phase structure—no explicit transition logic is defined, risking ambiguity.  
- **No stop condition:** Only the "comprehensive conclusion" phase mentions termination; earlier phases lack completion criteria.

**Instruction Quality:**  
- **Vague verbs:** "Judge which processes are completed" is unclear—no criteria for "completed" are provided.  
- **Mixed abstraction:** High-level workflow rules are mixed with implementation examples, creating redundancy.  
- **Overlapping scope:** The "important tip" duplicates phase 1’s "do not repeat" instruction.

**Control and Behavior:**  
- **Unclear uncertainty handling:** No guidance on how to proceed if phase status is ambiguous.  
- **Verbosity confusion:** Phase 2 demands "no thought process," yet the prompt includes explanatory examples.

**Input and Output Specification:**  
- **No input defaults:** Unclear what "current state" contains or how to handle missing state data.  
- **Output schema missing:** Only phase-specific formats are noted; no unified final output structure.

**Scope and Safety:**  
- **No error handling:** No instructions for tool failures or invalid transitions between phases.  
- **User authority ambiguity:** The model autonomously decides phase transitions without user confirmation.

**Efficiency and Maintainability:**  
- **Redundancy:** Examples restate rules already defined.  
- **Hidden assumptions:** Assumes "current state" reliably indicates phase completion.

**Summary:** The prompt defines a phased workflow but suffers from ambiguous transitions, unclear completion criteria, and redundancy. It needs explicit phase-completion rules, a consistent output schema, and error-handling measures.
