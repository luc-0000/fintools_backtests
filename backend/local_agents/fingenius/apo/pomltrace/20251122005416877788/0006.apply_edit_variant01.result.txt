# Critique

**Critique of the Prompt**

**1. Structural Issues**  
- **Missing goal**: The prompt lacks a clear, one-line definition of success (e.g., "produce a final conclusion report").  
- **No stop condition**: The only stop condition is calling `terminate`, but the prompt does not define when the model should transition between stages or what triggers the final stage.

**2. Instruction Quality**  
- **Vague verbs**: "判断哪些流程已经执行完毕" ("judge which processes are complete") is ambiguous—no criteria are given for determining completion.  
- **Mixed abstraction**: High-level workflow rules are mixed with implementation details (e.g., tool restrictions per stage), but stage transitions are not clearly defined.

**3. Control and Behavior**  
- **Unclear uncertainty handling**: The prompt does not specify how to handle missing or ambiguous state information (e.g., what if "current state" is undefined?).  
- **Verbosity confusion**: Some stages demand pure text output, but the prompt includes templated instructions that may encourage unnecessary formatting.

**4. Input and Output Specification**  
- **No input defaults**: The prompt assumes "current state" is provided but does not define what to do if it is missing or invalid.  
- **Output schema missing**: While example outputs are given, there is no consistent schema for responses across stages.

**5. Scope and Safety**  
- **No error handling**: The prompt lacks instructions for tool failures or invalid transitions (e.g., what if `terminate` is called prematurely?).  
- **User authority ambiguity**: The model must infer stage transitions based on state, but no explicit user approval is required for irreversible actions (e.g., moving to final stage).

**6. Efficiency and Maintainability**  
- **Redundancy**: The conditional warning about `terminate` is repetitive with the stage rules.  
- **Hidden assumptions**: It is assumed that "current state" includes stage and data status, but this is not explicitly defined.

**Summary**: The prompt has logical gaps in stage transitions, undefined completion criteria, and insufficient error handling. Clarify the goal, define state transitions explicitly, and specify fallback behaviors for missing inputs.
