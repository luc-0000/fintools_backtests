### PROMPT

根据当前状态，判断哪些流程已经执行完毕，直接执行下一个流程。如果流程前面已经执行过，不要重复执行。

**工作流程规则**：
1. **数据收集阶段**：  
   - 允许使用 `数据收集工具`，**禁用其他所有工具**（包括 `terminate`）。

   - 允许调用 `terminate` 结束任务。

### CRITIQUE

Based on the experiments and the original prompt template, here are specific issues and testable improvements:

- **Add explicit stage tracking variables** - The template lacks clear state management. Add variables like `current_stage: "data_collection" | "analysis" | "conclusion"` to enforce stage transitions.

- **Strengthen tool restriction enforcement** - The current conditional check only triggers after violation. Add proactive checks: `{{ if current_stage == "data_collection" }} 仅允许使用数据收集工具 {{ else if current_stage == "analysis" }} 禁止调用任何工具 {{ end }}`

- **Improve stage transition logic** - The template assumes implicit transitions. Add explicit triggers: `{{ if data_exists and current_stage == "data_collection" }} 立即进入深度分析阶段 {{ end }}`

- **Standardize output format per stage** - Add mandatory formatting: Data collection must output `[数据收集结果: {data}]`, analysis must use `[分析报告: {content}]`, conclusion must call terminate.

- **Clarify data existence checking** - Define what constitutes "already collected data" with specific variable names like `collected_data` to prevent ambiguity in state detection.

- **Add error recovery instructions** - Include fallback behavior: `{{ if违规操作 }} 立即停止工具调用，返回纯文本输出并说明"继续分析阶段" {{ end }}`

These changes will make stage transitions more deterministic and tool restrictions more enforceable.
